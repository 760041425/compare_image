# URD-001: Image Compare — 用户需求文档

## 版本: 0.1 (Draft)
## 日期: 2026-07-18

---

## 1. 用户画像

### 主要用户
- 需要从视频中查找特定图片内容的开发者/研究者
- 有基本的命令行操作能力
- 了解图片/视频的基本概念

## 2. 用户场景

### 场景 1: 以图搜帧
**前提**: 用户有一张查询图片和一段视频
**操作**: 用户指定图片和视频路径，系统自动比对
**期望结果**: 返回最相似的帧列表（按相似度排序），附带相似度分数

### 场景 2: 批量比对
**前提**: 用户有多张查询图片和多段视频
**操作**: 批量指定输入
**期望结果**: 每张图片的匹配结果分别输出

### 场景 3: 结果可视化
**前提**: 比对完成
**操作**: 查看结果
**期望结果**: 查询图片和匹配帧并排显示，标注相似度分数

## 3. 用户故事

| ID | 故事 | 优先级 |
|----|------|--------|
| US-001 | 作为用户，我想指定一张图片和一段视频，找到最相似的帧 | P0 |
| US-002 | 作为用户，我想调整相似度阈值，过滤低质量匹配 | P0 |
| US-003 | 作为用户，我想选择不同的比对算法 | P1 |
| US-004 | 作为用户，我想导出匹配结果（JSON/CSV） | P1 |
| US-005 | 作为用户，我想可视化查看匹配结果 | P1 |
| US-006 | 作为用户，我想控制帧提取的采样率 | P2 |

## 4. 约束

- 处理速度：1 分钟视频应在 30 秒内完成比对
- 内存：不超过 4GB
- 支持格式：图片 (JPG, PNG, BMP)；视频 (MP4, AVI, MKV)
